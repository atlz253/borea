## When to Use This Skill

Activate this skill when the user:

- Asks to plan a complex task or project
- Requests help organizing work before implementation
- Wants to break down a large feature into smaller components
- Needs assistance with task prioritization or scheduling
- Is starting a new development task and wants a structured approach

## How to Plan Tasks

### Step 1: Understand the Task
- Identify the main objective of the task
- Determine the expected outcome
- Clarify any requirements or constraints

### Step 2: Break Down into Steps
- Divide the task into logical, manageable sub-tasks
- Identify dependencies between steps
- Estimate time required for each step

### Step 3: Create Action Items
- List specific actions needed
- Assign priorities to tasks
- Identify resources or tools required

### Step 4: Review and Refine
- Check for missing elements
- Verify that all dependencies are accounted for
- Ensure the plan is realistic and achievable

## Planning Framework

When creating a task plan, use this structure:

1. **Task Overview**
   - Main objective
   - Expected deliverables
   - Success criteria

2. **Step-by-Step Breakdown**
   - List of sub-tasks in logical order
   - Dependencies between tasks
   - Estimated time for each

3. **Action Items**
   - Specific actions to be taken
   - Responsible parties (if applicable)
   - Resources needed

4. **Potential Challenges**
   - Risks or obstacles to anticipate
   - Mitigation strategies

## Guidelines

- Always start with understanding the complete scope of the task
- Break large tasks into smaller, implementable steps
- Consider dependencies and prerequisites for each step
- Make the plan actionable with specific deliverables
- Include time estimates where possible
- Review the plan for completeness before finalizing
